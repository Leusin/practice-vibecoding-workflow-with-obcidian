# [[할일앱]] — 에러 로그

## [2026-06-03] npm create vite 실행 시 "Operation cancelled"

**증상**
`npm create vite@latest . -- --template react` 실행 시 아무 메시지 없이 `Operation cancelled`로 종료됨.

**원인**
Vite는 대상 디렉토리가 비어있지 않으면 대화형 프롬프트를 띄운다. 그런데 `todo-app/` 안에 `.claude` 폴더가 이미 존재했고, 비대화형 환경에서는 프롬프트에 응답할 수 없어 자동으로 취소됨.

**해결**
`--overwrite` 옵션 추가로 기존 파일을 무시하고 강제 생성.
```bash
npm create vite@latest . -- --template react --overwrite
```

**교훈**
Claude Code가 작업하는 디렉토리에는 `.claude/` 폴더가 자동 생성된다. Vite 프로젝트를 빈 디렉토리에 만들 때도 이 폴더 때문에 "비어있지 않음" 판정이 날 수 있으므로 처음부터 `--overwrite`를 붙이는 것이 안전하다.

## [2026-06-03] Node.js / npm 미설치로 모든 명령 실패

**증상**
`npm` 명령 실행 시 `command not found: npm` 오류 발생. `node`, `brew`, `nvm`, `bun` 등 모두 없음.

**원인**
macOS 기본 설치 상태로, Node.js 관련 도구가 전혀 설치되지 않은 클린 환경이었음. 또한 Xcode Command Line Tools도 없어 일부 빌드 도구가 동작하지 않는 상태.

**해결**
아래 순서로 환경을 구성함.
1. `xcode-select --install` — Xcode CLT 설치 (팝업에서 확인 클릭, 약 10분 소요)
2. `curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash` — nvm 설치
3. `source ~/.zshrc && nvm install --lts` — Node.js LTS 설치 (v24.16.0)

**교훈**
Mac 신규 환경에서 React 개발을 시작할 때는 Xcode CLT → nvm → Node.js 순서가 표준 절차다. 특히 Xcode CLT는 nvm 스크립트 실행 중에도 요구될 수 있으므로 가장 먼저 설치한다.
