# 할일앱 — 기술 의사결정 기록

## 2026-06-03: 기술 스택 결정

### 결정 사항
- 프론트엔드: React
- 백엔드: 없음 (프론트엔드 only)
- 저장: localStorage
- 배포: Vercel

### 이유
- React: 학습 자료가 가장 풍부하고, 바이브코딩에서 가장 많이 사용됨
- 백엔드 없음: 첫 프로젝트이므로 복잡도를 최소화
- localStorage: 서버 없이 데이터를 유지하는 가장 간단한 방법
- Vercel: 무료이고, React 프로젝트 배포가 가장 간편

### 대안 (검토 후 기각)
- Vue.js: 좋은 프레임워크지만, React가 레퍼런스가 더 많음
- Supabase: DB가 필요하면 나중에 Phase 5로 추가 가능
