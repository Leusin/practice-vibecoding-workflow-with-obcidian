## 컴포넌트 구조

```
App
├── TodoInput      ← 할일 입력창 + 추가 버튼
└── TodoList       ← 할일 목록 전체
    └── TodoItem   ← 개별 할일 (체크박스 + 텍스트 + 삭제 버튼)
```

| 컴포넌트 | 역할 |
|---|---|
| `App` | 전체 상태(`todos`) 보관, localStorage 연동 |
| `TodoInput` | 텍스트 입력 받아서 App에 전달 |
| `TodoList` | `todos` 배열을 받아서 목록으로 렌더링 |
| `TodoItem` | 완료 토글, 삭제 버튼 처리 |

## 데이터 구조

할일 하나는 다음 세 가지 필드로 구성된다.

```js
{
  id: 1,           // 고유 식별자 (Date.now() 사용)
  text: "운동하기", // 할일 내용
  done: false      // 완료 여부
}
```

전체 상태는 이 객체들의 배열이다.

```js
todos = [
  { id: 1, text: "운동하기", done: false },
  { id: 2, text: "책 읽기",  done: true  },
]
```

## 데이터 흐름

상태(`todos`)는 `App`에 하나만 둔다. 모든 변경은 App의 함수를 통해서만 일어난다.

```
[추가] TodoInput → onAdd("운동하기") → App이 todos에 새 항목 추가
[완료] TodoItem  → onToggle(id)      → App이 해당 항목 done 반전
[삭제] TodoItem  → onDelete(id)      → App이 해당 항목 제거
```

- props는 항상 위(App)에서 아래(자식)로 내려준다.
- 이벤트는 함수(onAdd, onToggle, onDelete)로 위로 올린다.

### localStorage 연동

`App.jsx` 안에서 두 가지만 처리한다.

```js
// 앱 시작 시 — 저장된 데이터 불러오기
const [todos, setTodos] = useState(() => {
  const saved = localStorage.getItem("todos");
  return saved ? JSON.parse(saved) : [];
});

// todos가 바뀔 때마다 — 자동 저장
useEffect(() => {
  localStorage.setItem("todos", JSON.stringify(todos));
}, [todos]);
```

## 파일 구조

```
src/
├── components/
│   ├── TodoInput.jsx   # 입력창
│   ├── TodoList.jsx    # 목록
│   └── TodoItem.jsx    # 개별 항목
├── App.jsx             # 상태 관리 + localStorage
├── App.css             # 스타일
└── main.jsx            # 진입점 (건드릴 일 거의 없음)
```
